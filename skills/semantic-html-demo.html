What is Semantic HTML?

Open Project

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers. Instead of using generic <div> tags everywhere, semantic elements like <header>, <nav>, <article>, and <footer> provide structure and accessibility.

<!-- Non-Semantic (Less Clear) --> <div class="header"> <div class="nav"></div> </div> <div class="content"> <div class="article"></div> </div> <!-- Semantic (Better Practice) --> <header> <nav></nav> </header> <main> <article></article> </main> <footer></footer>

<header>

Represents introductory content, typically containing navigation, logos, or titles.

<nav>

Contains navigation links for the document or site.

<main>

Specifies the main content of the document - should be unique per page.

<article>

Represents independent content like blog posts, news articles, or user comments.

<section>

Groups related content together as a thematic section.

<aside>

Contains content tangentially related to main content, like sidebars.

<footer>

Represents footer content like copyright info, links, or contact details.

Benefits:

  • Better Accessibility - Screen readers understand the structure
  • Improved SEO - Search engines better understand content organization
  • Cleaner Code - More readable and maintainable
  • Better for Mobile - Proper structure helps responsive design
  • Future-Proof - Aligns with web standards
